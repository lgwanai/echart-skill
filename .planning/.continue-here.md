<decisions_made>

1. **Insight Engine uses pure DuckDB SQL** — no pandas dependency for core analysis, keeps it fast and dependency-light
2. **Report engine follows template-driven approach** — sections are built by routing insights to builders by slug, extensible for new templates
3. **Semantic models stored as JSON** — not a database, for git versioning and human editability
4. **Z-score threshold = 2.5** for anomaly detection (configurable)
5. **All engines follow the same pattern**: Python API + CLI + SKILL.md scenario, matching existing project conventions
