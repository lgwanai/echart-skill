  <name>Task 1: Rewrite DatabaseRepository for DuckDB</name>
  <files>database.py</files>
  <action>
    Rewrite database.py to use DuckDB instead of SQLite while preserving the public API:

    1. Replace `import sqlite3` with `import duckdb`
    2. Remove `from queue import Queue` â€” DuckDB doesn't need queue-based pooling the same way
    3. Rewrite `DatabaseRepository.__init__`:
       - Store `db_path` and `pool_size`
       - Create a list-based pool of DuckDB connections (not Queue, since DuckDB has single-writer model)
       - Use `threading.Lock` for write serialization instead of queue-based approach
    4. Rewrite `_initialize_pool`:
       - Use `duckdb.connect(self._db_path)` for each connection
       - Remove WAL mode PRAGMA (not applicable to DuckDB)
       - Remove `check_same_thread=False` (not needed for DuckDB)
       - Set DuckDB config: `conn.execute("SET threads=4")` and `conn.execute("SET memory_limit='2GB'")`
    5. Rewrite `connection()` context manager:
       - Acquire connection from pool with threading lock for write safety
       - Yield the connection
       - Return to pool on exit
    6. Update `execute_query()`:
       - Use `conn.execute(query, params).fetchdf()` then convert to list of dicts
       - DuckDB's execute returns DuckDBPyConnection, not cursor
    7. Update `execute_many()`:
       - Use `conn.executemany(query, params_list)` or loop with `conn.execute()`
       - DuckDB supports executemany for parameterized queries
    8. Update `close_all()`:
       - Close each DuckDB connection
    9. Update `get_repository()` singleton â€” change default db_path from "workspace.db" to "workspace.duckdb"
    10. Update all type hints: remove `sqlite3.Connection` references, use `duckdb.DuckDBPyConnection`

    Key differences from SQLite:
    - DuckDB's `conn.execute()` returns DuckDBPyConnection (has fetchdf(), fetchall(), fetchone())
    - No `conn.cursor()` needed â€” execute directly on connection
    - No PRAGMA commands â€” use SET for configuration
    - DuckDB creates the database file immediately on connect()
  </action>
