    Create `scripts/history_viewer.py` with three core functions, following the project's coding conventions (snake_case, type annotations, Chinese user-facing messages, structlog):

    **1. `view_import_history(db_path: str = "workspace.duckdb", limit: int = 20) -> str`**
    Query `_data_skill_meta` ordered by import_time DESC, limit rows. Format as markdown table:
    ```
    ## 导入历史

    | 文件名 | 表名 | 行数 | 导入时间 | 文件路径 |
    |--------|------|------|----------|----------|
    | sales.xlsx | sales | 1500 | 2026-04-11 10:30 | /path/to/sales.xlsx |
    ```
    - Use `COALESCE(row_count, 0)` for backward compatibility with records that don't have row_count
    - Truncate file_path to last 60 chars if longer (use `...` prefix)

    **2. `view_table_structure(db_path: str = "workspace.duckdb", table_name: str = None) -> str`**
    If table_name provided, show structure for that table. If None, list all user tables (exclude `_data_skill_meta` and tables starting with `_`).
    For each table, query `information_schema.columns` for column_name, data_type, is_nullable. Also get row count via `SELECT COUNT(*)`.
    Format as markdown table:
    ```
    ## 表结构: sales

    **行数:** 1500

    | 列名 | 类型 | 可空 |
    |------|------|------|
    | id | VARCHAR | YES |
    | name | VARCHAR | NO |
    ```

    **3. `view_table_relationships(db_path: str = "workspace.duckdb") -> str`**
    Query `_data_skill_meta WHERE parent_tables IS NOT NULL`. Parse JSON parent_tables, format as markdown table:
    ```
    ## 表关联关系

    | 目标表 | 来源表 | 行数 | 创建时间 |
    |--------|--------|------|----------|
    | merged_sales | sales_jan, sales_feb | 3000 | 2026-04-11 11:00 |
    ```
    If no relationships found, return "暂无表关联记录。"

    **Helper:** Add `format_markdown_table(headers: list[str], rows: list[list[str]]) -> str` to handle consistent table formatting with proper column alignment.

    Import pattern: use `sys.path.insert(0, ...)` for database import, same as other scripts. Use `DatabaseRepository` for connection pooling.
