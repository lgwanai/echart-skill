   ```python
   import argparse

   from dashboard_schema import DashboardConfig, ChartConfig, get_dashboard_json_schema
   from database import get_repository

   from server import ensure_server_running
   from chart_generator import get_baidu_ak
   ```

2. **Function: `load_dashboard_config(config_path: str) -> DashboardConfig`:**
   - Read JSON file from config_path
   - Parse and validate using DashboardConfig model
   - Return validated config
   - Log errors with structlog

3. **Function: `fetch_chart_data(chart: ChartConfig, db_path: str) -> pd.DataFrame`:**
   - Use get_repository(db_path)
   - Execute chart.query with pd.read_sql_query
   - Return DataFrame (may be empty)
   - Log warning if empty

4. **Function: `aggregate_map_scripts(charts: list[ChartConfig], option_jsons: list[str], base_url: str) -> tuple[str, str, str, str]`:**
   - Scan all echarts_option and custom_js for map usage
   - Return tuple: (bmap_script, china_script, world_script, province_scripts)
   - Reuse detection logic from chart_generator.py but iterate all charts
   - Use base_url for local asset paths

5. **Function: `generate_dashboard_html(config: DashboardConfig, output_path: str) -> str`:**
   - For each chart:
     - Fetch data with fetch_chart_data()
     - Build echarts_option (inject dataset if empty and df not empty)
     - Generate option_json
   - Aggregate all map scripts
   - Generate HTML with:
     - CSS Grid layout using config.columns, config.gap, config.row_height
     - Each chart container: `<div id="chart_{id}" style="grid-area: {row+1} / {col+1} / span {row_span} / span {col_span}"></div>`
     - Single `<script>` block initializing all charts
     - Window resize handler calling resize() on all chart instances
   - Write to output_path
   - Return output_path

6. **Function: `generate_dashboard(config_path: str, output_path: str | None = None) -> str`:**
   - Load config with load_dashboard_config()
   - Determine output_path (default: outputs/html/dashboards/{config.title}.html)
   - Call generate_dashboard_html()
   - Log success with output_path and access_url
   - Return output_path

7. **CLI `if __name__ == "__main__"` block:**
   - argparse with --config (required) and --output (optional)
   - Call generate_dashboard()
   - Handle exceptions with logger.error()

**CSS Grid Template:**
```html
<style>
    .dashboard-grid {{
        display: grid;
        grid-template-columns: repeat({columns}, 1fr);
        grid-auto-rows: {row_height}px;
        gap: {gap}px;
        padding: 16px;
    }}
    .chart-container {{
        min-height: {row_height}px;
        border: 1px solid #ddd;
        border-radius: 4px;
    }}
</style>
```

**Chart Container Pattern:**
```html
<div id="chart_{id}" class="chart-container" style="grid-row: {row+1} / span {row_span}; grid-column: {col+1} / span {col_span};"></div>
```

**ECharts Initialization Pattern:**
```javascript
var charts = [];
// For each chart:
(function(chartId, optionJson, customJs) {{
    var chart = echarts.init(document.getElementById(chartId));
    var option = optionJson;
    customJs;
    chart.setOption(option);
    charts.push(chart);
}})('chart_{id}', {option_json}, function() {{ {custom_js} }});
// Window resize:
window.addEventListener('resize', function() {{ charts.forEach(c => c.resize()); }});
```

Do NOT use iframe approach - single page with multiple ECharts instances.
Follow ensure_ascii=False pattern for JSON serialization.
Reuse map detection logic from chart_generator.py.
