  <action>
Create `scripts/server_cli.py` with three main commands:

**1. start_server() function:**
- Check if server already running using `check_server_running()` from server.py
- If running, load and return existing status
- If not running:
  - Find free port using `find_free_port()` (default range 8100-8200)
  - Start `scripts/server.py --daemon --port <port>` as subprocess
  - Wait up to 2 seconds for server to start (poll health endpoint)
  - Save status to `outputs/.server_status.json`:
    ```json
    {
      "pid": <process_id>,
      "port": <port>,
      "start_time": "<ISO timestamp>",
      "status": "running"
    }
    ```
- Return status dict

**2. stop_server() function:**
- Read status from `outputs/.server_status.json`
- If no status file or status != "running", return {"status": "not_running"}
- Send SIGTERM to process PID (use os.kill(pid, signal.SIGTERM))
- Wait up to 1 second for process to terminate
- Update status file: `{"status": "stopped", "stopped_time": "<ISO timestamp>"}`
- Return stopped status

**3. get_status() function:**
- Read `outputs/.server_status.json` if exists
- If file missing, return `{"status": "never_started"}`
- If status == "running", verify process still active using `os.kill(pid, 0)`
- If process dead, update status to "stopped" and return
- Return current status dict

**4. CLI interface with argparse:**
```bash
# Subcommands:
python scripts/server_cli.py start [--port PORT]
python scripts/server_cli.py stop
python scripts/server_cli.py status
```

**Error handling:**
- Port conflicts: auto-retry next port in range
- Permission errors: return clear error message
- Timeout: report timeout status

**Output format:**
- All commands return JSON to stdout
- Human-readable messages to stderr for logging
- Exit codes: 0=success, 1=error, 2=timeout

Use imports from server.py:
```python
from server import check_server_running, find_free_port, ServerLifecycle
```

DO NOT duplicate code from server.py - import and reuse.
