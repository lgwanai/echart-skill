# Agent Report & Dashboard Workflows

This directory defines how the Agent should create enterprise reports and dashboards.
The final report/dashboard should be authored by the Agent from these Markdown
specifications, not generated by a fixed Python renderer.

Python scripts may be used only as data tools:

- import/clean/query local data
- profile tables
- run SQL aggregation
- run insight/forecast/attribution helpers
- validate generated standalone HTML

The final narrative, layout, industry framing, visual hierarchy, and chart
selection must be decided by the Agent according to the workflow specs.

## Files

- `report_workflow.md` — enterprise report workflow and required reasoning steps
- `dashboard_workflow.md` — dashboard planning and construction workflow
- `visual_templates/light.md` — light visual direction for Agent-authored output
- `visual_templates/dark.md` — dark visual direction for Agent-authored output

## Non-goals

- Do not add a Python report/dashboard generator for final rendering.
- Do not encode industry playbooks as Python if/else rules.
- Do not force one fixed HTML structure for all businesses.
